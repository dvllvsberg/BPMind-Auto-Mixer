
Техническое задание для BPMind Auto Mixer
Общая задача
Необходимо разработать MVP настольного приложения BPMind Auto Mixer.
Приложение должно автоматически анализировать локальную музыкальную библиотеку пользователя и создавать непрерывные музыкальные сессии с автоматическим подбором треков и переходов.

Приоритет разработки
Разработка должна идти поэтапно.
Не пытаться сразу реализовать весь функционал.

Приоритет:
Этап 1
Сканирование библиотеки.
Этап 2
Анализ треков.
Этап 3
Построение последовательности треков.
Этап 4
Воспроизведение.
Этап 5
Автоматические переходы.
Этап 6
Улучшение качества анализа.

Рекомендуемый стек
Язык:
Python 3.12+
Почему:

большое количество аудио-библиотек;
быстрое прототипирование;
удобная работа с DSP.


Интерфейс Предпочтительно:
PySide6
Причины:
нативный интерфейс;
высокая производительность;
проще интеграция с Python.
Electron использовать только при наличии серьёзных причин.

Хранение данных
Использовать:
SQLite
Не использовать:
JSON как основную базу данных
JSON допустим только для настроек.

Сканирование файлов
После выбора папки:
1.  Найти все поддерживаемые аудиофайлы.
2.  Добавить их в базу.
3.  Проверить:
путь;
размер;
дату изменения.

Если файл уже анализировался и не менялся:
не выполнять повторный анализ


Анализ BPM
Для MVP использовать готовые библиотеки.
Подходящие варианты:

librosa
aubio
essentia

Определять:
BPM

и сохранять результат в базу.


Анализ громкости
Для каждого трека определить:
средняя громкость и пиковая громкость.


Анализ структуры трека
На первом этапе не пытаться находить:

куплеты;
припевы;
дропы.

Это усложняет MVP.
Вместо этого определить:

энергия по времени

Например:

0-10 сек
10-20 сек
20-30 сек

и т.д.

На основе этого строить карту трека.


Точки переходов
Для MVP искать:

тихие участки;
резкие падения энергии;
начало аутро.

Каждый найденный участок сохранять как:

TransitionCandidate


Модель данных трека

Пример:

Track:
    id
    path
    title
    artist

    duration

    bpm

    loudness_avg
    loudness_peak

    key

    transition_candidates

    analyzed_at


Генератор микса
Создать отдельный модуль:

MixGenerator

На вход получает:

список треков
режим запуска

На выходе:

MixSession


Логика выбора следующего трека
Основной фактор:

BPM

Дополнительные:

громкость
тональность

Каждому кандидату присваивать рейтинг.
Выбирать лучший вариант.
Не использовать полностью случайный выбор.


Энергия сета

Генератор должен поддерживать:

Calm
Peak
Wave
Random

На MVP можно реализовать только:

Calm
Peak


Mix Session
Пример структуры:

MixSession:

    tracks

    transitions

    start_mode

    created_at


Воспроизведение
Для MVP достаточно:

Play
Pause
Next
Previous


Smart Skip
При нажатии Next:

Попытаться выполнить короткий переход.
Если невозможно:
выполнить обычное переключение.


Переходы
Начать только с:

Crossfade

Не реализовывать сразу:

Echo;
Sweep;
Reverse FX;
Impact FX.

Сначала добиться стабильной работы обычного кроссфейда.

Только потом добавлять остальные эффекты.


Производительность
Цель MVP:

Библиотека:

1000 треков

должна корректно анализироваться и работать на обычном домашнем ПК.



Что НЕ делать в MVP
Не реализовывать:

Spotify;
SoundCloud;
YouTube Music;
облачные сервисы;
AI-рекомендации;
жанровые классификаторы;
профессиональные DJ-функции;
экспорт миксов;
мобильную версию.

Сначала необходимо получить рабочий локальный автомиксер.
