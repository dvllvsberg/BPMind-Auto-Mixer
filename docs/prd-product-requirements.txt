# BPMind Auto Mixer

## Описание проекта

Необходимо создать настольное приложение для автоматического построения музыкальных миксов на основе локальной библиотеки пользователя.

Проект вдохновлён Pioneer MIXTRAX, однако не является его копией.

Главная цель программы — автоматически создавать непрерывные музыкальные сессии из локальной коллекции треков без участия пользователя.

Пользователь не должен вручную сводить треки, настраивать BPM, использовать деки или работать как DJ.

Программа должна самостоятельно:

* анализировать музыку;
* выбирать порядок воспроизведения;
* подбирать переходы;
* создавать ощущение непрерывного DJ-сета.

---

# Основная идея

Пользователь выбирает папку с музыкой.

Программа:

1. Сканирует библиотеку.
2. Анализирует треки.
3. Создаёт Mix Session.
4. Автоматически воспроизводит её с переходами между треками.

Цель проекта — не идеальное DJ-сведение, а создание ощущения живого непрерывного микса.

---

# Архитектура

Проект должен состоять из двух независимых частей.

## Core Engine

Отвечает за:

* анализ аудио;
* построение базы треков;
* генерацию микса;
* подбор переходов;
* управление воспроизведением.

Движок не должен зависеть от интерфейса.

---

## Desktop Application

Отвечает за:

* выбор папки;
* отображение процесса анализа;
* управление воспроизведением;
* отображение текущего и следующего трека;
* настройки.

---

# MVP

Первая версия должна работать полностью оффлайн.

Не использовать:

* Spotify;
* SoundCloud;
* Apple Music;
* внешние API;
* облачные сервисы.

Поддерживаемые форматы:

* MP3
* FLAC
* WAV
* M4A

---

# Анализ библиотеки

Для каждого трека необходимо определить:

Обязательно:

* BPM;
* длительность;
* среднюю громкость.

Желательно:

* тональность;
* потенциальные точки переходов.

Все результаты анализа должны сохраняться локально.

Повторное сканирование уже проанализированных файлов не выполнять без необходимости.

---

# Двухэтапное сканирование

## Быстрый анализ

Выполняется сразу.

Определяет:

* BPM;
* длительность;
* громкость.

После завершения быстрого анализа пользователь уже может начать воспроизведение.

---

## Глубокий анализ

Выполняется в фоне.

Определяет:

* тихие участки;
* падения энергии;
* возможные точки переходов;
* предполагаемые интро;
* предполагаемые аутро.

Результаты глубокого анализа используются для повышения качества миксов.

---

# Mix Session

Mix Session — это конечный автоматически построенный музыкальный сет.

Это не бесконечное радио.

После генерации формируется последовательность треков и переходов между ними.

После окончания сессии пользователь решает, что делать дальше.

---

# Режимы запуска

## Random Mix

Случайный стартовый трек.

Дальнейшая последовательность строится автоматически.

---

## Start From Track

Пользователь вручную выбирает первый трек.

Остальной маршрут строится автоматически.

---

## Start Calm

Поиск трека с относительно низкой энергией.

Сессия строится по принципу:

спокойно → разгон → пик → спад.

---

## Start Peak

Поиск трека с высокой энергией.

Сессия начинается сразу с активной части.

---

# Подбор треков

Основной параметр:

* BPM.

Дополнительные параметры:

* громкость;
* тональность (если определена).

Жанры не являются обязательным параметром.

Пользователь обычно сам выбирает библиотеку определённого жанра.

---

# Кривая энергии

Микс не должен иметь постоянную интенсивность.

Предпочтительное поведение:

Спокойный участок
↓
Разгон
↓
Пик
↓
Спад
↓
Новый разгон
↓
Новый пик

Поведение должно напоминать живой DJ-сет.

---

# Философия переходов

Главная задача переходов — создать ощущение непрерывности.

Не обязательно идеально сводить треки.

Важно, чтобы слушатель не воспринимал переход как резкое переключение.

---

# Типы переходов

Примеры:

* Crossfade;
* Echo Out;
* Sweep;
* Reverse FX;
* Impact Transition;
* Hard Cut FX.

Тип перехода выбирается автоматически.

---

# Момент перехода

Запрещается использовать фиксированное правило.

Нельзя делать:

* всегда на последних 15 секундах;
* всегда на 80% длительности трека;
* всегда на аутро.

Программа должна выбирать точки перехода динамически.

Желательно использовать:

* тихие участки;
* падения энергии;
* аутро;
* участки, удобные для сведения.

Разные треки должны играть разное количество времени.

Один трек может быть воспроизведён почти полностью.

Другой может использоваться только частично.

Поведение должно казаться естественным и непредсказуемым.

---

# Управление

Минимально необходимы:

* Play;
* Pause;
* Next;
* Previous.

---

# Smart Skip

При нажатии Next допускается короткий переходный эффект вместо мгновенного переключения.

Переход должен выглядеть осознанным.

---

# Завершение Mix Session

Когда микс подходит к концу, показать пользователю окно выбора.

Варианты:

1. Продолжить воспроизведение в случайном порядке без микширования.
2. Пересобрать новый микс.
3. Завершить воспроизведение после окончания текущей сессии.

---

# Важные ограничения

Не пытаться реализовать профессиональный DJ-софт.

Не использовать деки.

Не использовать DJ-контроллеры.

Не реализовывать ручное сведение.

Не реализовывать профессиональные функции Serato, Rekordbox или Traktor.

Проект ориентирован на обычного пользователя.

Главная кнопка программы должна выглядеть примерно как:

"Выбрать папку → Построить микс → Слушать музыку"

Чем меньше действий требуется от пользователя, тем лучше.
