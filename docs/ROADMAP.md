## MVP / ТЗ (этапы 1–6)

| Этап | Статус |
|------|--------|
| 1. Сканирование библиотеки (SQLite, инкремент) | ✅ |
| 2. Анализ BPM, громкость, energy map | ✅ |
| 3. MixGenerator, режимы Calm / Peak / Wave / Random / from_track | ✅ |
| 4. Play / Pause / Next / Prev | ✅ |
| 5. Автоматические переходы | ✅ TEA, 8+ профилей |
| 6. Улучшение качества анализа | 🟡 groove, auto-profile |

Стек (Python, PySide6, SQLite) и структура `app/` + `engine/` — по ТЗ.

**Сознательно за пределами MVP ТЗ:** экспорт WAV/MP3, portable exe, расширенный GUI.

## RPD (продукт)

| Требование | Статус |
|------------|--------|
| Оффлайн, локальная папка | ✅ |
| Быстрый + глубокий анализ | ✅ |
| Mix Session — конечный сет | ✅ |
| BPM + громкость в подборе | ✅ + groove |
| Тональность в подборе | 🟡 поле в модели, анализ/scoring нет |
| Динамические точки перехода | ✅ |
| Переходы только автоматически (TEA) | ✅ |
| Smart Skip на Next | ✅ v1.7.0 |
| Окно конца сета (3 варианта RPD) | ✅ v1.7.1 — заранее, не после остановки |
| UX «папка → микс → слушать» | 🟡 v1.6 компактнее, настройки отдельно |

---

## TEA (Transition Engine)

| Пункт | Статус |
|-------|--------|
| Отдельный этап после Mix Builder | ✅ `mix_pipeline.py` |
| `decide(context)`, cooldown | ✅ |
| Все базовые профили из TEA-дока | ✅ |
| Составные цепочки (Echo → Filter → …) | ❌ отложено |
| Расширенный контекст (key, drop, vocal…) | ❌ будущее |

Подробнее: [transition-engine.md](transition-engine.md).

## MVP Rules

| Правило | Статус |
|---------|--------|
| Без внешних API / AI / DJ-deck | ✅ |
| Качество микса важнее количества фич | 🟡 цель, ручная подкрутка ещё бывает |
| Стабильный crossfade и BPM | ✅ |
